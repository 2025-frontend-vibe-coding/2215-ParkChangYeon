// 1번째 프롬프트
이번 서비스는 게시판 서비스입니다. 게시판CRUD 페이지 및 로그인, 회원가입 페이지의 UI/UX를 제작하고, 백엔드와 연동해주세요.
백엔드는 기존에 만들어둔 @파일을(를) 참고하여 다음 사항에 반영해 주세요

1. 화면구성
1-1. 헤더
 - 서비스 로고(서비스 이름도 괜찮음), 로그인 버튼, 회원가입 버튼을 표시
 - 로고(이름)은 좌측에 로그인 버튼과 회원가입 버튼은 한 묶음으로 우측에 정렬

1-2. 메인 페이지
 - 시작화면으로 할것
 - 헤더와 화면으로 구성
 - 화면에는 전형적인 커뮤니티 느낌
 - 중앙에 생성된 글을 보여주고, 글쓰기 버튼을 표시
 - 글은 제목과 내용을 구분하여 표시
 - 글의 제목 우측에 글쓴이를 표시
 - 글의 우측에는 ⁝을 만들어, 클릭 시에 글 수정과 글 삭제 버튼을 표시
 - 글 수정 버튼 클릭 시, 글 수정 페이지로 이동
 - 글 삭제 버튼 클릭 시, 백엔드에 삭제 요청
 - 글 목록은 백엔드에서 글 조회를 Pageable로 처리했으니 한번에 10개씩만 표시
 - Pagenation은 글 목록 하단에 위치
 - 비회원(로그인하지 않은 경우)일 시, 글 조회와 글 쓰기 기능을 사용 불가하고, 오류 메시지 출력

1-3. 글 쓰기 및 글 수정 페이지
 - 비회원인 유저가 억지로 들어왔을 시, 로그인 페이지로 이동
 - 헤더와 화면으로 구성
 - 글 쓰기와 글 수정 페이지는 글을 쓸 경우와 수정할 경우의 입력해야할 값이 같으므로 컴포넌트화하여 중복코드를 예방
 - 글 제목, 글 내용을 받을 것
 - 글 작성(글 쓰기) 버튼을 클릭 시 백엔드에 요청

1-4. 로그인 페이지
 - 헤더와 화면으로 구성
 - 유저의 id와 비밀번호를 받음
 - 로그인 버튼을 클릭 시 백엔드에 요청
 - 로그인 성공 시 메인 페이지로 이동
 - 로그인 폼 하단에 "계정이 없으신가요? 회원가입" 이런 문구 추가 및 회원가입 문구 클릭 시 회원가입 페이지로 이동

1-5. 회원가입 페이지
 - 헤더와 화면으로 구성
 - id, 이메일, 닉네임, 비밀번호를 받음
 - 회원가입 버튼을 클릭 시 백엔드에 요청
 - 회원가입 성공 시 로그인 페이지로 이동
 - 회원가입 폼 하단에 "계정이 있으신가요? 로그인" 이런 문구 추가 및 로그인 문구 클릭 시 로그인 페이지로 이동

2. 디자인 요구사항
2-1. 전체적인 디자인 요구사항
 - 헤더는 상단에 고정
 - 헤더를 제외한 페이지 상단에 페이지 제목을 표시
 - 모바일에서도 잘 보이는 방응형 디자인
 - 색의 베이스는 흰색으로 하되, 추가적인 색상은 조화롭게 섞어서 사용가능
 - 전체적으로 부드럽고, 부담스럽지 않은 애니메이션 효과
 - 로딩 시 애니매이션 호과
 - 글 목록 로딩 시 글에 회색 슬라이드 효과
 - 버튼 호버시 마우스 위치에 따른 슬라이드 효과
 - 입력폼은 페이지 중앙에 표시
 - 입력폼은 label과 input의 한 묶음
 - label 밑에 input 위치
 - input에는 placeholder가 있어야 함
 - 입력폼은 세로로 나열
 - 입력폼이 있을 시 이에 대한 버튼이라면 항상 폼 아래에 버튼을 위치

2-2. 메인페이지
 - 글 목록은 중앙에 배치
 - 각 글은 세로로 나열
 - 글 호버 시 호버된 글 배경을 살짝 짙게 변경
 - Pagenation은 좌우 화살표 사이에 페이지 숫자

2-3. 글 쓰기 및 글 수정 페이지
 - 페이지 상단, 페이지 제목 좌측에 '<'와 같이 이전 페이지로 가는 버튼 표시

3. 기술요구사항
 - React + TypeScript사용
 - Next.js APP Router사용
 - Tailwind CSS로 스타일링
 - 입력폼은 react-hook-form 사용
 - Redux Toolkit으로 로그인 상태관리
 - Shadcn/ui 컴포넌트 활용

4. 기능요구사항
 - 비회원 서비스 지원 안함
 - TypeScript를 사용하되, any타입은 지양
 - 반복되는 코드 사용을 지양하기위해 컴포넌트화 적극활용

5. 오류처리(예외처리)
 - 회원가입시에 닉네임은 최대 15자 제한 
 - 입력이 너무 길 경우 길이 제한을 안내
 - 회원가입시에 입력 형식을 지킬것(예: example@gmail.com)
 - 없는 게시글을 삭제하고자 할 경우, 경고창 표시
 - 작성자 이외의 유저가 게시글 수정을 시도할 경우, 경고창 표시
 - 없는 게시글을 수정하고자 할 때, 경고창 표시
 - 없는 게시글 조회를 시도했을 때, 경고창 표시
 - 이외의 백엔드에서 처리 후 넘기는 모든 예외를 처리할 것
